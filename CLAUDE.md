# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

**Arrow Game** — Unity 6 (6000.4.1f1) mobile puzzle game, 2D URP. Company: Azgalord. Target platform: iOS.

All game scripts live in `Assets/_Game/Scripts/`. The only scene is `Assets/Scenes/SampleScene.unity`.

## Unity-specific development notes

A **unity-mcp** MCP server is connected, exposing tools: `Unity_RunCommand`, `Unity_GetConsoleLogs`, `Unity_Camera_Capture`, `Unity_SceneView_Capture2DScene`, `Unity_AssetGeneration_*`. Use these to inspect the Editor state, read console output, or capture scene screenshots without leaving Claude Code. The Unity Editor must be open for these to work.

There is no CLI build command — the project is opened and run inside the **Unity Editor**. To build or run tests:
- Open the project in Unity Hub using the exact editor version `6000.4.1f1`
- Play mode: press the Play button in the Editor
- Build: File → Build Settings → iOS
- Tests (com.unity.test-framework is included): Window → General → Test Runner

`Library/`, `Temp/`, and `Logs/` are excluded from git and auto-generated by Unity on first open. Never commit them.

## Architecture

### Core loop

```
GridManager (MonoBehaviour, scene singleton)
  ├── WinEffect (component, same GameObject) — win screen + confetti
  ├── BloomSetup (auto-added in Awake if missing) — URP Bloom post-processing
  └── spawns ArrowCell GameObjects as children at runtime
        └── ArrowHead (child GameObject)
              ├── MeshRenderer + MeshFilter  — procedural triangle mesh
              ├── PolygonCollider2D — tap hitbox
              └── ArrowHeadTap (MonoBehaviour) — relays OnMouseDown → ArrowCell.TryMove()

HeartsManager (separate scene GameObject, [ExecuteAlways])
  └── builds GameCanvas at runtime (header bar: 3 hearts left + "Level X" centre label)
```

### Key classes (`Assets/_Game/Scripts/`)

**`ArrowData`** (plain C# class, defined in `ArrowCell.cs`)  
Data-only: `List<Vector2Int> cells` (ordered path, tail→head) + `ArrowDirection exitDirection`. The last cell is the arrowhead position.

**`ArrowCell`** (`ArrowCell.cs`)  
Represents one entire snake arrow. Initialized via `Init(ArrowData, GridManager, Sprite, Color)` — never through Awake/Start. Root transform is fixed at world `(0,0,0)`; the `LineRenderer` uses local space so a single `transform.position +=` in `MoveOffScreen()` slides both the line and the arrowhead child together. Collision failure triggers `ShakeAnimation` + `FlashCollision` + `HeartsManager.Instance?.LoseHeart()`. Also has `OnMouseDown` directly on the root so tapping the body or the head both call `TryMove()`. Body has an `EdgeCollider2D` (radius = `cellSize * 0.18f`) for the tap hitbox along the line. `MoveOffScreen()` re-samples a growing point list each frame to produce a smooth snake-trail animation before `Destroy`ing the GameObject.

**`ArrowHeadTap`** (also in `ArrowCell.cs`)  
Thin relay component placed on the ArrowHead child. Only purpose: `OnMouseDown() → owner.TryMove()`.

**`GridManager`** (`GridManager.cs`)  
Owns `ArrowCell[,] grid` — each cell stores a reference to the `ArrowCell` that occupies it (multiple cells of the same snake share the same reference). `Awake()` initializes the grid, auto-adds `BloomSetup` if absent, and centers the GridManager transform. `static int currentLevel` survives scene reloads (incremented by `WinEffect.Advance()`). Key methods:
- `Reinitialize(cols, rows, cs)` — reset grid at runtime before placing arrows (used by `TestLevel`)
- `PlaceArrow(ArrowData)` — validates, spawns, registers all cells
- `ClearArrow(ArrowCell)` — nulls all cells of a snake, fires `WinEffect.Show()` if grid empty
- `IsPathClear(ArrowData)` — walks from arrowhead cell in exit direction; returns `true` if no other `ArrowCell` reference is hit before the grid edge

**`HeartsManager`** (`HeartsManager.cs`)  
`[ExecuteAlways]` singleton — visible in Editor without Play. Builds the screen-space Canvas (header bar with 3 hearts + level label) via `RebuildCanvas()` each time `OnEnable` / `OnValidate` fires. `LoseHeart()` decrements hearts and calls `SceneManager.LoadScene` to restart after all 3 are lost. `RuntimeInit()` is called by `TestLevel.Start()` to set the static `Instance` and refresh the level label.

**`WinEffect`** (`Assets/_Game/Scripts/UI/WinEffect.cs`)  
Attached to the GridManager GameObject. `Show(levelNumber)` starts a coroutine that bursts confetti (procedural quad meshes) from world origin, then sequences in "Level X" → "Solved!" → "Next →" button with scale/fade animations. "Next →" calls `Advance()` which increments `GridManager.currentLevel` and reloads the scene.

**`BloomSetup`** (`Assets/_Game/Scripts/UI/BloomSetup.cs`)  
Auto-added by `GridManager.Awake()`. Creates a global URP `Volume` with `Bloom` override and enables post-processing on the main camera. `DontDestroyOnLoad` so it survives level reloads.

**`BackgroundSetup`** (`Assets/_Game/Scripts/UI/BackgroundSetup.cs`)  
`[ExecuteAlways]` — renders a full-screen world-space quad at `camera.z + 10` using an assigned material (e.g. a radial-gradient shader). Sorting order -100. Rebuilt on `OnEnable`/`OnValidate`; destroyed on `OnDisable`/`OnDestroy`. Assign `bgMaterial` in Inspector.

**`HintButton`** (`Assets/_Game/Scripts/UI/HintButton.cs`)  
`[ExecuteAlways]` — builds a pill-shaped HINT button (border + pill + lamp icon + TMP label) anchored to the bottom of a `ScreenSpaceCamera` canvas. The pill sprite is generated procedurally with 9-slice rounding. `OnHintClicked()` currently only `Debug.Log`s — hint logic is not yet implemented. Assign `lampSprite` in Inspector (falls back to `Resources/lamp`).

**`TestLevel`** (`TestLevel.cs`)  
Temporary level loader; calls `gm.Reinitialize(9, 9, 0.5f)` then `PlaceArrow(...)`. Uses `#if UNITY_EDITOR` to load a fast 2-arrow level in the Editor and a full 10-arrow 9×9 puzzle in builds. Replace with a real level system later.

### Coordinate system

- Grid origin (col 0, row 0) = bottom-left
- `GridManager.transform.position` = `(-(columns-1)*cellSize/2, -(rows-1)*cellSize/2, 0)` — centered on world origin
- `GetWorldPosition(col, row)` = `transform.position + (col*cellSize, row*cellSize, 0)`

### Execution order dependency

`GridManager.Awake()` allocates the grid. `TestLevel.Start()` calls `PlaceArrow`. This is safe because Unity guarantees all `Awake()` calls complete before any `Start()`. **Do not move grid allocation back to `Start()`.**

### Rendering / HDR glow

- `LineRenderer` is on the ArrowCell root, `useWorldSpace = false`, material uses `Sprites/Default`
- Arrowhead is a procedural triangle mesh (not a sprite); points **right** at 0° rotation; `ExitRotation()` in `ArrowCell` rotates it: Up=90°, Down=-90°, Left=180°, Right=0°
- Sorting order: line = 1, arrowhead MeshRenderer = 2
- **HDR color** = `new Color(r * glowIntensity, g * glowIntensity, b * glowIntensity, 1)` — values > 1 trigger URP Bloom. `glowIntensity` (default 1.5) is set on `GridManager` and `HeartsManager`

### Inspector setup

**GridManager GameObject** also needs `WinEffect` component attached.  
- `Arrow Head Sprite` — legacy field, arrowhead is now a procedural mesh (field ignored)
- `Arrow Colors` — array of colors cycled per arrow
- `Grid Settings` — columns, rows, cellSize
- `Arrow Appearance & Feel` — `arrowSpeed` (units/sec), `arrowBodyWidth` (fraction of cellSize), `arrowHeadScale` (fraction of cellSize)
- `Collision Feedback` — `collisionFlashColor`, `collisionFlashDuration`
- `Glow Intensity` — HDR multiplier (1 = no glow, 3 = strong neon)

**HeartsManager GameObject** — separate scene object; assign `Heart Sprite` in Inspector (falls back to a procedural circle if unassigned).

**BackgroundSetup GameObject** — assign `bgMaterial` (e.g. a radial-gradient shader material) in Inspector.

**HintButton GameObject** — assign `lampSprite` in Inspector; tweak `buttonColor`, `borderColor`, `contentColor`, `bottomOffset`, `glowIntensity`.
